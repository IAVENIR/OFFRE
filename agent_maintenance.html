Comprendre la différence entre les 3 chemins
A · Azure OpenAI API

On appelle directement le modèle GPT via l'API Azure. Pas de licence par utilisateur — les techniciens accèdent à l'agent via la plateforme web existante. On paie uniquement les tokens consommés (quelques centimes par question). C'est le chemin le plus économique et le plus flexible, mais il nécessite un développement backend léger.

B · Copilot Studio

L'agent est construit dans l'interface no-code Microsoft. La licence builder est gratuite, mais chaque réponse de l'agent consomme des Copilot Credits — vendus en packs prépayés de 25 000 à $200/mois. C'est le « carburant » : la licence ouvre l'atelier, les crédits alimentent la machine. Plus facile à construire, mais plus cher à l'usage à grande échelle.

C · ChatGPT Business

Chaque technicien a besoin d'un compte ChatGPT Business à $25/mois pour accéder au Custom GPT. L'usage est ensuite inclus dans la licence. C'est le chemin le plus simple (zéro développement), mais le plus coûteux : $5 000/mois fixes pour 200 users, même si l'agent n'est utilisé qu'une fois par jour. Pertinent uniquement si les licences sont déjà en place pour d'autres usages.

Prise de décision
Questions clés à clarifier ensemble
1
API Kimoce : quels endpoints sont disponibles ? Recherche full-text dans les commentaires, ou filtres structurés uniquement ?
2
Volume de données : combien d'interventions dans Kimoce ? Quelle profondeur historique ? Quel débit API supporté ?
3
Qualité des données : les comptes rendus techniciens sont-ils structurés et exploitables, ou très hétérogènes ?
4
Documentation technique : volume, formats, centralisation (SharePoint, serveur fichiers) ? Besoin réel des techniciens sur les notices ?
5
Droits d'accès : un technicien doit-il voir uniquement les interventions de son périmètre (site, zone, contrat) ?
6
Environnement Azure : le client dispose-t-il d'un tenant Azure actif ? Contraintes d'hébergement France / souveraineté ?
7
Licences existantes : quel plan ChatGPT est en place (Business, Enterprise, Pro) ? Licences M365 Copilot actives ?
8
Référent interne : qui maintiendra l'agent (mise à jour docs, prompts) ? Quel niveau de transfert de compétences souhaité ?
